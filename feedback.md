## [권장] P1. 키보드만으로 파일 업로드가 불가능 (접근성)
- **위치**: CSS L75 `input[type=file]{display:none;}` · HTML L169 `<div class="dropzone" id="dropzone">` · JS L1001–1003
- **현상**: 업로드 트리거가 **클릭 핸들러만 있는 `<div>`** 이고, 실제 `<input type=file>`은 `display:none`이라 탭 순서에서도 빠집니다. 화면에 키보드로 도달 가능한 업로드 버튼이 **하나도 없어**, 키보드·스크린리더 사용자는 파일을 올릴 방법이 없습니다 (WCAG 2.1.1 Keyboard 위반).
- **제안**: 드롭존을 버튼처럼 만들어 Enter/Space로 파일 선택이 열리게 합니다.
  ```html
  <div class="dropzone" id="dropzone" role="button" tabindex="0"
       aria-label="CSV 파일 선택 (Enter 또는 Space)">
  ```
  ```js
  dz.addEventListener("keydown", e => {
    if (e.key === "Enter" || e.key === " ") { e.preventDefault(); fi.click(); }
  });
  ```
  - `:focus-visible` 외곽선 스타일도 함께 주면 포커스 위치가 보입니다.

## [권장·판단필요] P2. 표준편차가 표본 기준(n−1) — 한국 고교 교육과정은 모집단 기준(÷n)
- **위치**: `calculateStats()` L446 `variance = ... /(n - 1)` → L447 `stddev`
- **현상**: 분산을 **(n−1)** 로 나눕니다(표본표준편차). 그러나 한국 고등학교 「확률과 통계」의 분산·표준편차 정의는 **Σ(xᵢ−m)²/n** (모집단 식, ÷n)입니다. 학생이 배운 공식으로 손계산한 값과 이 도구의 값이 **체계적으로 어긋납니다** (n=10이면 약 +5.4%).
- **판단 포인트**: 측정값을 "표본"으로 보면 n−1도 통계학적으로 타당합니다. 다만 **대상이 고교 수업**이고 학생이 직접 계산·대조할 가능성이 높다면 교육과정 정의(÷n)에 맞추는 편이 혼란이 적습니다.
- **제안(교육과정 정합을 택할 경우)**:
  ```js
  const variance = n > 0 ? vals.reduce((a,b)=>a+(b-mean)**2,0)/n : 0;
  ```
  - 그대로 표본(n−1)을 유지한다면 **사용자 결정 사항**이며 수정 불필요. 결정만 회신해 주세요.

## [참고] 의도 확인만 필요 (수정 불필요 가능성 높음)

### N1. '초당 평균 변화율'이 x축 단위(분) 선택과 무관하게 항상 `/초`
- **위치**: `renderStats()` L649 — `"...+ "/초"`. 정의상 "초당" 변화율이라 값 자체는 정확합니다. x축을 '분'으로 보는 학생에게 단위가 엇갈려 보일 수 있으나, **명확히 '초당'이라 라벨돼 있으므로 오류 아님.** 그대로 둬도 무방.

### N2. 센서값 열 자동 선택이 인덱스/카운터 열을 고를 수 있음
- **위치**: `detectColumns()` L354–359 — 시간 열이 아닌 **첫 번째 숫자 열**을 센서값으로 선택.
- 보드 CSV에 '번호/샘플 인덱스' 같은 숫자 열이 센서값보다 앞에 있으면 그 열이 잡힐 수 있습니다. 다만 **수동 '센서값 열 선택' 드롭다운(L203)으로 즉시 교정 가능**하고 헤더명 제안(L921–924)도 있어 실사용 위험은 낮음 → **현행 수용 가능.**

### N3. `parseFloat` 관용 파싱
- **위치**: `cleanRows()` L396, `detectColumns()` L357 — `parseFloat("12abc")→12`, `parseFloat("1,234")→1`(쉼표 앞에서 끊김).
- 이지메이커 보드 CSV는 보통 순수 숫자라 영향 적음. 천 단위 쉼표가 든 값이 들어올 가능성이 있으면 방어가 필요하나, **일반적 입력에선 불필요** → 과한 처리 지양 차원에서 현행 유지 권장.

## 검증 완료 (정상 — 수정 금지)
- **CSV 파서**(`parseCSV`): UTF-8 BOM 제거, 따옴표 내부 쉼표·이스케이프(`""`) 처리, 빈 줄 무시 — 정확.
- **시간 파싱**(`parseTimeToSeconds`): `HH:mm` / `HH:mm:ss` / `.SSS` 허용, h>23·min>59·s>59 검증 — 정확.
- **자정 넘김 보정**(`cleanRows` L399–403): 직전 시각보다 역행하면 +86400초로 경과시간 연속 — 타당.
- **통계 로직**: 평균/중앙값(짝수개 평균)/최대·최소 및 발생시점/범위/전체변화량/초당변화율/최대변화구간 — 모두 정확. n=1 분기(변화율·분산 0 처리)도 안전.
- **그래프**(`drawChart`): 동일값일 때 yMin/yMax ±1 처리, 격자·축·마커 화면밖 보정, 점 ≤300개만 표시 — 정상.
- **교육적 안전 설계**: 단위 미상 시 '센서값' 고정, 해석/질문/복사요약 전부 **원인·물리량을 단정하지 않고 '가능성'으로 표현** — 설계 의도 충실.
- **오류 처리·안내**: 비CSV/빈파일/헤더만/데이터0개/1개·결측행 제외에 대한 메시지(err·warn·ok) 적절.
- **런타임 안정성**: `DOMContentLoaded` 후 이벤트 바인딩, `$` 헬퍼, null 참조 경로 없음. 열 변경·단위 변경·센서종류 변경·시간단위 변경 시 재렌더 경로 정상.
- **보안**: 전 구간 `textContent`/`createElement` 사용 → HTML 주입 불가. 파일은 브라우저 내 처리(서버 미전송) 명시.
